# 减脂方案模板

## 方案选择流程

初始化新用户时，在收集完基础数据后，向用户介绍以下方案并让其选择：

```
我们有两种减脂饮食方案：

A. 凯圣王模式（碳水循环，适合有训练基础、想稳定保肌减脂）
B. 谭成义模式（极端碳水循环，适合有经验、追求快速减脂、执行力强）

不确定的话告诉我你的训练基础和体型，我帮你推荐。
```

推荐逻辑：
- 训练基础 < 6个月 / 新手 → 推荐 A
- 训练基础 ≥ 6个月 + 执行力强 + 能接受极低碳日 → 可选 B
- 体脂 > 20% → 优先 B（快速减脂效果更明显）

---

## 方案 A：凯圣王模式

通过温和的碳水波动欺骗代谢，保肌减脂，适合大多数有训练基础的人。

### 体型判断

| 体型 | 特征 | 碳水基准 |
|------|------|---------|
| 内胚型（易胖） | 容易囤脂，减脂慢 | 体重kg × 2g |
| 外胚型（易瘦） | 不容易长肉，代谢快 | 体重kg × 3g |

### 计算基准

```
碳水：内胚型 2g/kg，外胚型 3g/kg
蛋白质：1g/kg（固定）
脂肪：内胚型 0.8g/kg
```

### 一周碳水总量分配

```
周碳水总量 = 体重kg × 碳水基准 × 7天

高碳日（2天）= 周总量 × 50% ÷ 2  → 安排在最大训练日（腿日）
中碳日（3天）= 周总量 × 35% ÷ 3  → 安排在普通训练日
低碳日（2天）= 周总量 × 15% ÷ 2  → 安排在休息日
```

### 示例：72kg 内胚型男性

```
周碳水总量 = 72 × 2 × 7 = 1008g

高碳日碳水 = 1008 × 50% ÷ 2 = 252g
中碳日碳水 = 1008 × 35% ÷ 3 = 118g
低碳日碳水 = 1008 × 15% ÷ 2 = 76g

蛋白质（每天）= 72g
脂肪：高碳日 36g / 中碳日 58g / 低碳日 86g
```

### 碳日映射规则

| 训练内容 | 碳日类型 |
|---------|---------|
| 腿日 | 高碳日 |
| 胸日 / 背日 | 中碳日 |
| 肩日 / 手臂 / 有氧 | 中碳日 |
| 休息日 | 低碳日 |

### 核心原则

- 低碳日不是零碳日，脂肪要补上来，总热量不能崩太低
- 高低碳日总热量差距不超过 500 kcal
- 蛋白质每天固定，不随碳日变化

### 代谢停滞应对

连续 3 天体重不动 + 训练状态变差 → 安排一次重启高碳日（碳水 × 1.5 倍，持续 1 天）

## 方案 B：谭成义模式

极端碳水循环，4天极低碳 + 1天极高碳，快速减脂，执行要求高。

### 计算基准

```
极低碳日（4天）：
  碳水 = 体重kg × 0.5g 以内
  蛋白质 = 体重kg × 1.5g
  脂肪 = 体重kg × 0.8g

极高碳日（1天）：
  碳水 = 体重kg × 4~6g（补碳，重启代谢）
  蛋白质 = 体重kg × 1g
  脂肪 < 20g/天（极低，严格控制）
```

### 周循环节奏

```
极低碳日 × 4天 → 极高碳日 × 1天 → 循环
```

安排建议：
- 极高碳日安排在最大训练日（腿日）或感觉最疲惫的那天
- 极低碳日可以有训练，但强度可能下降，属正常现象

### 示例：72kg 男性

```
极低碳日：碳水 36g，蛋白质 108g，脂肪 58g → 约 1050 kcal
极高碳日：碳水 360g（按5g/kg），蛋白质 72g，脂肪 <20g → 约 1880 kcal
```

### 碳日映射规则

| 日类型 | 碳日类型 |
|--------|---------|
| 最大训练日（腿日）或第5天 | 极高碳日 |
| 其余训练日 + 休息日 | 极低碳日 |

### 核心原则

- **严禁欺骗餐**：极高碳日只吃干净低GI碳水（红薯、燕麦、糙米），不是放纵日
- 极高碳日脂肪必须降至极低（< 20g），碳水和脂肪不能同时高
- 极低碳日热量会很低，短期疲惫感正常，坚持到补碳日会恢复

### 适用食物（极高碳日）

| 食物 | 每100g碳水 | 备注 |
|------|-----------|------|
| 红薯（熟） | 20g | 首选，低GI |
| 燕麦（干） | 66g | 低GI，饱腹 |
| 糙米饭（熟） | 23g | 低GI |
| 白米饭（熟） | 28g | 可用，GI较高 |

## 方案写入档案

选定方案后，将以下信息写入 MEMORY.md：

```
饮食方案：[A/B]
方案名称：[凯圣王模式/谭成义模式]
每日目标：
  方案A → 高碳日：碳水Xg/蛋白质Xg/脂肪Xg；中碳日：...；低碳日：...
  方案B → 极低碳日：碳水Xg/蛋白质Xg/脂肪Xg；极高碳日：...
```

---

## 碳日预排（两种方案均需执行）

### 初始化时执行

收集训练计划后，根据方案推算未来 2~4 周每天的碳日类型，**批量写入饮食记录表**。

每条记录只填：日期、碳日类型、碳水目标、热量目标（餐次和食物留空，等用户当天填）。

### 批量写入示例

用 `feishu_bitable_app_table_record` action=batch_create，一次写入 7~14 天：

```json
{
  "records": [
    {"fields": {"日期": 1742400000000, "碳日类型": "中碳日", "碳水目标（g）": 118, "热量目标（kcal）": 2100}},
    {"fields": {"日期": 1742486400000, "碳日类型": "高碳日", "碳水目标（g）": 252, "热量目标（kcal）": 2400}},
    {"fields": {"日期": 1742572800000, "碳日类型": "低碳日", "碳水目标（g）": 76, "热量目标（kcal）": 1800}}
  ]
}
```

方案 B 的碳日类型字段值用：`极低碳日` / `极高碳日`（需在建表时加入这两个选项）。

### 动态调整

- 用户临时换训练日 → 更新对应日期的碳日类型
- 出现代谢停滞（方案A）→ 将下一个低碳日改为高碳日
- 每 2 周滚动预排下 2 周的碳日计划
